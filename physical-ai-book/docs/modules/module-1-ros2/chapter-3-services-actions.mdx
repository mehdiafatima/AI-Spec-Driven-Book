# Chapter 3: Services, Actions, and Parameters

This chapter expands on ROS 2 communication patterns by introducing **Services**, **Actions**, and **Parameters**. These mechanisms provide ways for nodes to interact synchronously, handle long-running goals, and dynamically configure behavior.

## ROS 2 Services: Synchronous Request/Reply

**Services** are a synchronous communication method in ROS 2, primarily used for request/reply interactions between nodes. When a client node calls a service, it sends a request and waits for a response from the service server node.

### Key Characteristics:

-   **Synchronous**: The client blocks until it receives a response or a timeout occurs.
-   **Request/Reply**: A single request triggers a single response.
-   **Unary Operation**: Best suited for operations that can be completed quickly and return a definitive result.

### When to use Services:

-   Triggering a specific action (e.g., "take a picture," "move arm to home position").
-   Querying information (e.g., "get robot status," "get map data").
-   Configurations that require a confirmation.

### Defining a Custom Service

Service definitions are stored in `.srv` files within a ROS 2 package, typically in a `srv/` directory.

Example (`AddTwoInts.srv`):

```
int64 a
int64 b
---
int64 sum
```
The `---` separates the request fields from the response fields.

### Creating a Simple ROS 2 Service (Python)

Let's create a service server that adds two integers and a client that calls this service.

1.  **Create a package and `srv` directory**:
    ```bash
    cd ~/ros2_ws/src
    ros2 pkg create --build-type ament_python my_service_pkg
    mkdir -p my_service_pkg/srv
    ```
2.  **Define the service (`my_service_pkg/srv/AddTwoInts.srv`)**:
    ```
    int64 a
    int64 b
    ---
    int64 sum
    ```
3.  **Edit `setup.py` and `package.xml`**:
    -   **`package.xml`**: Add these lines:
        ```xml
        <build_depend>rosidl_default_generators</build_depend>
        <exec_depend>rosidl_default_runtime</exec_depend>
        <member_of_group>rosidl_interface_packages</member_of_group>
        ```
    -   **`setup.py`**: Add `my_service_pkg/srv/AddTwoInts.srv` to `data_files` and import `rosidl_default_generators`.
        ```python
        from setuptools import setup
        import os
        from glob import glob

        package_name = 'my_service_pkg'

        setup(
            name=package_name,
            version='0.0.0',
            packages=[package_name],
            data_files=[
                ('share/ament_index/resource_index/packages',
                    ['resource/' + package_name]),
                ('share/' + package_name, ['package.xml']),
                (os.path.join('share', package_name, 'srv'), glob('srv/*.srv')), # Add this line
            ],
            install_requires=['setuptools'],
            zip_safe=True,
            maintainer='your_name',
            maintainer_email='you@example.com',
            description='TODO: Package description',
            license='TODO: License declaration',
            tests_require=['pytest'],
            entry_points={
                'console_scripts': [
                    'add_two_ints_server = my_service_pkg.add_two_ints_server:main',
                    'add_two_ints_client = my_service_pkg.add_two_ints_client:main',
                ],
            },
        )
        ```
4.  **Create Service Server (`my_service_pkg/my_service_pkg/add_two_ints_server.py`)**:
    ```python
    import rclpy
    from rclpy.node import Node
    from my_service_pkg.srv import AddTwoInts # Import your custom service

    class AddTwoIntsService(Node):

        def __init__(self):
            super().__init__('add_two_ints_server')
            self.srv = self.create_service(AddTwoInts, 'add_two_ints', self.add_two_ints_callback)
            self.get_logger().info('AddTwoInts Service Server Ready.')

        def add_two_ints_callback(self, request, response):
            response.sum = request.a + request.b
            self.get_logger().info(f'Incoming request: a={request.a}, b={request.b}')
            self.get_logger().info(f'Sending response: sum={response.sum}')
            return response

    def main(args=None):
        rclpy.init(args=args)
        node = AddTwoIntsService()
        rclpy.spin(node)
        rclpy.shutdown()

    if __name__ == '__main__':
        main()
    ```
5.  **Create Service Client (`my_service_pkg/my_service_pkg/add_two_ints_client.py`)**:
    ```python
    import sys
    import rclpy
    from rclpy.node import Node
    from my_service_pkg.srv import AddTwoInts # Import your custom service

    class AddTwoIntsClient(Node):

        def __init__(self):
            super().__init__('add_two_ints_client')
            self.cli = self.create_client(AddTwoInts, 'add_two_ints')
            while not self.cli.wait_for_service(timeout_sec=1.0):
                self.get_logger().info('service not available, waiting again...')
            self.req = AddTwoInts.Request()

        def send_request(self, a, b):
            self.req.a = a
            self.req.b = b
            self.future = self.cli.call_async(self.req)
            rclpy.spin_until_future_complete(self, self.future)
            return self.future.result()

    def main(args=None):
        rclpy.init(args=args)
        client = AddTwoIntsClient()
        if len(sys.argv) != 3:
            client.get_logger().info(f'Usage: ros2 run {sys.argv[0]} a b')
            return
        response = client.send_request(int(sys.argv[1]), int(sys.argv[2]))
        client.get_logger().info(f'Result of add_two_ints: for {client.req.a} + {client.req.b} = {response.sum}')
        client.destroy_node()
        rclpy.shutdown()

    if __name__ == '__main__':
        main()
    ```
6.  **Build and Source**:
    ```bash
    cd ~/ros2_ws
    colcon build --packages-select my_service_pkg
    source install/setup.bash
    ```
7.  **Run (separate terminals)**:
    -   Terminal 1 (Server): `ros2 run my_service_pkg add_two_ints_server`
    -   Terminal 2 (Client): `ros2 run my_service_pkg add_two_ints_client 5 7`

## ROS 2 Actions: Long-Running Goals with Feedback

**Actions** are a communication type designed for long-running tasks that provide periodic feedback and allow for preemption (cancelling a goal). They are built on top of topics and services.

### Key Characteristics:

-   **Asynchronous Goal-Oriented**: Client sends a goal, and the action server works on it.
-   **Feedback**: Server provides continuous updates on the progress of the goal.
-   **Result**: Server provides the final outcome of the goal.
-   **Preemption**: Client can cancel the goal at any time.

### When to use Actions:

-   Robot navigation (e.g., "go to a specific pose").
-   Complex manipulation sequences (e.g., "pick up object X and place it at Y").
-   Any task that takes a significant amount of time and for which intermediate progress is useful.

### Defining a Custom Action

Action definitions are stored in `.action` files within a ROS 2 package, typically in an `action/` directory.

Example (`Fibonacci.action`):

```
int64 order
---
int64[] sequence
---
int64[] partial_sequence
```
The first `---` separates the goal fields from the result fields. The second `---` separates the result fields from the feedback fields.

### Creating a Simple ROS 2 Action (Conceptual)

Implementing a full action server and client is more involved, but the structure is as follows:

1.  **Define Action**: Create a `.action` file (e.g., `my_action_pkg/action/Fibonacci.action`).
2.  **Modify `package.xml` and `setup.py`**: Similar to services, you need to add build and execution dependencies for `rosidl_default_generators` and include the `.action` file in `data_files` within `setup.py`.
3.  **Action Server Node**:
    -   Creates an action server for the custom action.
    -   Implements a callback for new goal requests.
    -   Publishes feedback periodically.
    -   Publishes the final result when the goal is completed or canceled.
4.  **Action Client Node**:
    -   Creates an action client for the custom action.
    -   Sends a goal request to the server.
    -   Receives and processes feedback.
    -   Receives and processes the final result.
    -   Can send a preemption request.

## ROS 2 Parameters: Dynamic Configuration

**Parameters** allow you to make nodes more configurable without recompiling the code. They are key-value pairs that can be set at node startup or dynamically changed during runtime.

### Key Characteristics:

-   **Dynamic**: Values can be read and written during a node's execution.
-   **Node-specific**: Each node has its own set of parameters.
-   **Type-safe**: Parameters have defined types (e.g., `string`, `int`, `double`, `bool`, arrays).

### When to use Parameters:

-   Robot configuration (e.g., PID gains, motor limits, sensor offsets).
-   Algorithm tuning (e.g., detection thresholds, navigation costs).
-   Debugging flags.

### Using Parameters (Python)

1.  **Declare Parameters in a Node**:
    ```python
    import rclpy
    from rclpy.node import Node

    class ParameterNode(Node):

        def __init__(self):
            super().__init__('parameter_node')
            self.declare_parameter('my_int_param', 10) # Declare with default value
            self.declare_parameter('my_string_param', 'hello')

            # Get parameters
            my_int = self.get_parameter('my_int_param').value
            my_string = self.get_parameter('my_string_param').value
            self.get_logger().info(f'Initial parameters: int={my_int}, string={my_string}')

            # Update a parameter
            self.set_parameters([rclpy.parameter.Parameter('my_int_param', rclpy.Parameter.Type.INTEGER, 20)])

            my_int = self.get_parameter('my_int_param').value
            self.get_logger().info(f'Updated int parameter: {my_int}')

    def main(args=None):
        rclpy.init(args=args)
        node = ParameterNode()
        rclpy.spin(node)
        node.destroy_node()
        rclpy.shutdown()

    if __name__ == '__main__':
        main()
    ```
2.  **Run and Inspect Parameters**:
    ```bash
    # Run the node
    ros2 run my_package parameter_node

    # List parameters of a node
    ros2 param list /parameter_node

    # Get a parameter value
    ros2 param get /parameter_node my_int_param

    # Set a parameter value (can also be done at node launch)
    ros2 param set /parameter_node my_string_param "new_value"
    ```

## Conclusion

Services, Actions, and Parameters provide essential tools for building sophisticated ROS 2 applications. Services are for immediate request-response, Actions for complex long-running tasks with feedback, and Parameters for flexible runtime configuration.
